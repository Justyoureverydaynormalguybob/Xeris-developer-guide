1. Understanding the Xeris Architecture

Triple Consensus Protocol

Xeris solves the blockchain trilemma by combining three consensus layers into a single unified protocol:

Proof of Work (Light-PoW)

Unlike legacy chains requiring massive energy consumption, Xeris uses a "Light-PoW" model. Your node contributes computational availability simply by staying online — no GPU farms needed.

Proof of Stake (PoS)

Validators must stake a minimum of 100 XRS to participate. Your Validation Power (VP) is calculated as: VP = Stake × Uptime. Higher VP = higher reward share.

Proof of History (PoH)

The chain maintains a verifiable sequence of events through cryptographic timestamps. Nodes verify blockhashes to ensure tamper-proof transaction ordering.

3. Setting Up Your Dev Environment

Prerequisites

  • Node.js (v18+) — For building web-based dApps
  • Rust (latest stable) — For validator nodes or native tools
  • curl — For testing API calls
  • Git — For cloning repos

Option A: Web dApp Developer (Recommended)

# Create a new project
mkdir my-xeris-dapp && cd my-xeris-dapp
npm init -y

# Install dependencies
npm install axios         # HTTP client for API calls
npm install tweetnacl     # Cryptography (Ed25519 signing)
npm install bs58          # Base58 encoding (addresses)

Option B: Running a Validator Node

# Clone the XerisCoin repository
git clone https://github.com/ZZachWWins/xeriscoin.git
cd xeriscoin

# Build the node
cargo build --release

# Generate validator keys
./target/release/wallet keygen

# Start your validator
./target/release/xrs-node --validator 138.197.116.81 miner.json ledger.dat

Back up miner.json immediately! This file contains your private key. No entity under the Xeris Network Foundation can recover lost keys.

Option C: Mobile Development (Termux/Android)

pkg install nodejs git curl rust
# Then follow Option A or B above

8. Building Your First dApp

Step 1: Xeris API Client

A reusable JavaScript class that wraps all Xeris endpoints:

class XerisClient {
  constructor() {
    this.API = "http://138.197.116.81:50008";
    this.NET = "http://138.197.116.81:56001";
  }

  // REST API
  async getStats() {
    return (await fetch(`${this.API}/v2/stats`)).json();
  }

  async getAccount(addr) {
    return (await fetch(`${this.API}/v2/account/${addr}`)).json();
  }

  async getBlocks(limit = 10) {
    return (await fetch(`${this.API}/v2/blocks?limit=${limit}`)).json();
  }

  async getValidators() {
    return (await fetch(`${this.API}/v2/validators`)).json();
  }

  async search(query) {
    return (await fetch(`${this.API}/v2/search?q=${query}`)).json();
  }

  // JSON-RPC
  async rpc(method, params = []) {
    const res = await fetch(`${this.API}/rpc`, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ jsonrpc:"2.0", id:1, method, params })
    });
    return (await res.json()).result;
  }

  async getBalance(a) { return this.rpc("getBalance", [a]); }
  async getLatestBlockhash() { return this.rpc("getLatestBlockhash"); }
  async sendTransaction(tx) { return this.rpc("sendTransaction", [tx]); }

  // Network API (port 56001)
  async airdrop(addr, amt) {
    return (await fetch(`${this.NET}/airdrop/${addr}/${amt}`)).json();
  }

  async submitTx(tx) {
    return (await fetch(`${this.NET}/submit`, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ tx_base64: tx })
    })).json();
  }

  async stake(pubkey) {
    return (await fetch(`${this.NET}/stake`, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ pubkey: pubkey })
    })).json();
  }
}

Step 2: Use It

const client = new XerisClient();

// Get network stats
const stats = await client.getStats();
console.log(`Block Height: ${stats.blockHeight}`);

// Check a wallet balance
const balance = await client.getBalance("YOUR_ADDRESS");
console.log(`Balance: ${balance.value} lamports`);

// Get recent blocks
const blocks = await client.getBlocks(5);
blocks.forEach(b => console.log(`Slot ${b.slot}`));

9. Wallet Integration

The Xeris Wallet app supports WalletConnect via the Reown protocol, enabling QR-code wallet connections.

Chain ID Format (CAIP-2)

solana:<GenesisHash>

Reown AppKit Setup

import { createAppKit } from "@reown/appkit";

const xerisChain = {
  id: "solana:YOUR_GENESIS_HASH",
  name: "Xeris L1",
  nativeCurrency: { name: "XerisCoin", symbol: "XRS", decimals: 9 },
  rpcUrls: {
    default: { http: ["http://138.197.116.81:50008/rpc"] }
  }
};

const appKit = createAppKit({
  projectId: "YOUR_REOWN_PROJECT_ID",
  chains: [xerisChain],
  metadata: {
    name: "My Xeris dApp",
    description: "Built on Xeris",
    url: "https://your-dapp.com"
  }
});

Note: Coordinate with the wallet developer on the exact genesis hash. The current placeholder is solana:CustomGenesisHash1234567890 — both the dApp and wallet must use the same chain ID.

10. Transaction Handling

Sending a Transaction

import nacl from 'tweetnacl';

async function sendXRS(keypair, recipient, amount) {
  const client = new XerisClient();

  // 1. Get recent blockhash
  const { blockhash } = await client.getLatestBlockhash();

  // 2. Construct transaction
  const tx = {
    recentBlockhash: blockhash,
    instructions: [{
      from: keypair.publicKey,
      to: recipient,
      amount: amount
    }]
  };

  // 3. Sign
  const msg = Buffer.from(JSON.stringify(tx));
  const sig = nacl.sign.detached(msg, keypair.secretKey);

  // 4. Submit
  return client.submitTx(Buffer.from(sig).toString('base64'));
}

Transaction serialization format may evolve. Check the latest XerisCoin GitHub repo for the most current format.

Validators must stake a minimum of 100 XRS. Rewards are based on:

Validation Power (VP) = Stake Amount × Uptime Percentage

Developer Tips

  1. Start simple — Build a wallet checker before tackling complex dApps
  2. Use the REST API first — It's easier than JSON-RPC for reading data
  3. Back up your keys — miner.json is unrecoverable if lost
  4. Test with the faucet — Use /airdrop to get testnet XRS first
  5. Check GitHub regularly — New features and endpoints are added frequently
  6. CORS note — Browser dApps may need a proxy server for API calls depending on node CORS config
